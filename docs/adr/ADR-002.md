# ADR-002: Modular monolith core + microservices at the periphery

**Status:** Accepted (2026-07-02) | **Decider:** Chief Architect / Product Owner
**Source:** SAD Tom 1, General Decision GD-002

## Decision
Modular monolith core + microservices at the periphery.

## Rejected alternatives
Pure microservices from day one (the previous draft had 22 empty service folders)

## Rationale and consequences
A small team cannot operate 80 microservices. The core (ontology, IAM, workflow, CDE, forms, notifications, reporting) is one Go modular monolith with strict package boundaries. Heavy domains (BIM conversion, AI, TBM telemetry, CPM engine, analytics, GIS, integrations, mobile sync) are separate services. Core packages are extracted into services only when load demands it.
