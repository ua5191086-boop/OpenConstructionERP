# ADR-001: Ontology-first core (Foundry-style semantic layer)

**Status:** Accepted (2026-07-02) | **Decider:** Chief Architect / Product Owner
**Source:** SAD Tom 1, General Decision GD-001

## Decision
Ontology-first core (Foundry-style semantic layer).

## Rejected alternatives
Classic modular ERP (Odoo-style) where each module owns its tables

## Rationale and consequences
An infrastructure project is a single graph of objects (ring -> work package -> tunnel -> contract -> payment). Modular ERPs lose connectivity at module boundaries — the main pain of Primavera+SAP+Aconex zoos. All entities are ontology objects with types, properties, links, actions and full bitemporal version history. Modules are applications over the shared graph.
