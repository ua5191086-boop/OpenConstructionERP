# ADR-006: Event backbone: Apache Kafka (domain events, permanent retention) + NATS (realtime/mobile sync)

**Status:** Accepted (2026-07-02) | **Decider:** Chief Architect / Product Owner
**Source:** SAD Tom 1, General Decision GD-006

## Decision
Event backbone: Apache Kafka (domain events, permanent retention) + NATS (realtime/mobile sync).

## Rejected alternatives
RabbitMQ (was in the previous draft alongside Kafka and NATS)

## Rationale and consequences
Event sourcing of domain events is mandatory: audit for banks/auditors, replay for AI training, as-of-date delay analysis for free. Three brokers is operational suicide; RabbitMQ removed.
